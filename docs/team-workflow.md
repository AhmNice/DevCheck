# 🤝 Team Workflow (2 Developers)

## 1️⃣ Create Issue

- Feature or bug must start as an issue
- Define acceptance criteria

## 2️⃣ Create Branch

git checkout main
git pull origin main
git checkout -b feature/issue-name

## 3️⃣ Develop

- Keep changes small
- Follow folder structure
- Write clean commits

## 4️⃣ Push & Open PR

- Fill PR template
- Add screenshots if UI

## 5️⃣ Review

- Reviewer checks logic, structure, types
- Request changes if needed

## 6️⃣ Merge

- After approval only
- Delete branch
- Pull updated main locally

---

## Golden Rules

- No direct pushes to main
- No large PRs
- No emotional code reviews
- Discuss major changes before coding
